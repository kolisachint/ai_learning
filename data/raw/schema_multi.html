Data Warehouse Schema — Marketing Domain

This document describes the BigQuery table schemas for the marketing analytics domain. Three tables are defined: campaigns, ad_impressions, and campaign_conversions.

Campaign Master Data

One row per marketing campaign. Tracks campaign metadata and budget.

campaigns
name type mode description
campaign_id STRING REQUIRED Unique campaign identifier (UUID)
name STRING REQUIRED Campaign display name
channel STRING NULLABLE Channel: search | social | display | email | affiliate
objective STRING NULLABLE awareness | consideration | conversion | retention
budget_usd NUMERIC NULLABLE Total campaign budget in USD
spent_usd NUMERIC NULLABLE Amount spent to date in USD
is_active BOOLEAN NULLABLE Whether the campaign is currently running
target_audienceJSON NULLABLE Audience targeting criteria as JSON
start_date DATE REQUIRED Campaign start date (UTC)
end_date DATE NULLABLE Campaign end date; null if ongoing
created_at TIMESTAMP REQUIRED Record creation timestamp (UTC)

Ad Impression Log

One row per ad impression served to a user. High-volume table — partitioned by served_at.

name type mode description
impression_id STRING REQUIRED Unique impression identifier
campaign_id STRING REQUIRED FK to campaigns table
ad_id STRING REQUIRED Creative / ad unit identifier
user_id STRING NULLABLE Authenticated user; null if anonymous
device_type STRING NULLABLE desktop | mobile | tablet | ctv
placement STRING NULLABLE Ad placement position on the page
was_clicked BOOLEAN NULLABLE True if the impression resulted in a click
cost_usd NUMERIC NULLABLE Cost of this impression in USD (CPM/CPC)
served_at TIMESTAMP REQUIRED UTC timestamp when the ad was served

Table: campaign_conversions

Attribution records linking a conversion event back to a campaign impression.

name type mode description
conversion_id STRING REQUIRED Unique conversion record ID
campaign_id STRING REQUIRED FK to campaigns table
impression_id STRING NULLABLE FK to ad_impressions; null for view-through
user_id STRING NULLABLE FK to users table
conversion_type STRING REQUIRED signup | purchase | lead | app_install
revenue_usd NUMERIC NULLABLE Revenue value of the conversion in USD
attribution_modelSTRING NULLABLE last_click | first_click | linear | data_driven
converted_at TIMESTAMP REQUIRED UTC timestamp when the conversion occurred
