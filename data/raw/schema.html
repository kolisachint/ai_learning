BigQuery Schema: payments

This table records every payment transaction processed through the platform.

name type mode description
payment_id STRING REQUIRED Unique payment identifier (UUID v4)
order_id STRING REQUIRED FK to orders table
customer_id STRING REQUIRED FK to customers table
method STRING NULLABLE Payment method: card | bank_transfer | wallet | crypto
provider STRING NULLABLE Payment provider: stripe | paypal | adyen | braintree
amount NUMERIC REQUIRED Payment amount before fees
fee NUMERIC NULLABLE Transaction fee charged by the provider
currency STRING NULLABLE ISO 4217 currency code
status STRING NULLABLE pending | authorised | captured | failed | refunded
is_refund BOOLEAN NULLABLE True if this record is a refund
reference_id STRING NULLABLE Provider transaction reference
metadata JSON NULLABLE Arbitrary payment metadata from provider
initiated_at TIMESTAMP REQUIRED UTC timestamp when payment was initiated
settled_at TIMESTAMP NULLABLE UTC timestamp when funds were settled
